Recursion Diagnostic (Time 10 minutes)

  • This is my first point

  • Begin with a diagnostic: write a function that produces the sum of n positive integers, Give teachers 5 minutes to circulate

    Use factorial to control cognitive load. Factorial is objectively easier. Math notation hides the recursive structure; in this case we can use this.

    Pogil for recursion using factorial

    Show examples. Teachers identify the base case and recursive call Factorial using Recursion Pogil Sum

    Fibonacci (define (fib n) (cond ((= n 0) 0) ((= n 1) 1) (else (+ (fib (- n 1)) (fib (- n 2))))))

    Given an example evaluate an application of the example

    Write multiplication as a recursion of addition (define (mul a b) (if (= b 0) 0 (+ a (mul a (sub1 b)))))

    Write exponentiation as a recursion of multiplication (define (exp a b) (if (= b 0) 1 (* a (mul a (sub1 b)))))

    Think about how we teach multiplication already. This is how we teach it, as a abstract recursive procedure.

    Write Newtown’s approximation takes in a function and a delta and recursively finds the y-intercept of the derivative taken at a random point on that function until y-int miunus the point is less than delta. Point slope formula, subtract original, unless your done call Newton on the difference. (define (newt f x) (let* ((delta 0.01) (x1 x) (x2 (+ x delta)) (y1 (f x1)) (y2 (f x2)) (slope (/ (- y2 y1) (- x2 x1))) (y-intercept (- y1 (* slope x1))) (x-intercept (/ (* -1 y-intercept) slope))) (cond [(< (abs (- x-intercept x)) delta) x-intercept] [else (newt f x-intercept)])))
